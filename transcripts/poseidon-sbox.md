**What happened**: The LLM's first attempt used `linarith`, which works over ordered
types (`ℕ`, `ℤ`, `ℝ`) but not `ZMod p`. The Lean kernel rejected it. On the second
attempt, the LLM read the error, switched to `linear_combination` (which works over
any commutative ring), and the proof went through.

The final proof is 5 lines:
1. Extract `q = x * x` from the first constraint
2. Extract `r = q * q` from the second constraint
3. Extract `y = r * x` from the output constraint
4. Rewrite the goal using all three
5. `ring` closes `x * x * (x * x) * x = x ^ 5`

Total wall time: ~45 seconds (dominated by two `lake build` invocations).
